# Security policy

## Reporting a vulnerability

On GitHub, open this repository’s **Security** tab → **Report a vulnerability** (private reporting). That keeps details private until a fix is ready.

If Advisories are unavailable, open a **private** issue only if the repo owner has enabled that workflow, or contact the maintainers through a channel they publish in the README.

Please include: affected version/commit, reproduction steps, and impact (data exposed, RCE, etc.).

## Scope

In scope: this codebase, default Docker Compose layout, and documented configuration.

Out of scope: third-party services (Telegram, LLM providers, MongoDB Atlas, etc.) — report those to the respective vendors.

## After you report

Maintainers will acknowledge when they can, triage severity, and coordinate a fix and release note. There is no SLA; this is volunteer-maintained software unless stated otherwise in the README.
